# 食材随机选择器 - 产品需求文档

## 1. 项目概述

### 项目名称
**吃什么** - 食材随机选择器

### 项目背景
帮助用户从海量食材库中随机选择一道菜，支持多维度筛选（菜系、用餐时段、食材类型），解决"今天吃什么"的选择困难症。

### 目标用户
- 日常家庭烹饪者
- 餐厅经营者
- 美食爱好者

### 核心价值
- 海量食材库（成千上万个食材）
- 多级分类筛选
- 一键随机选择
- 多平台支持（网页 + 小程序）

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 食材选择
- **随机选择**：从当前筛选结果中随机选中一款食材
- **重新选择**：保留筛选条件，再次随机选择
- **清空选择**：清除当前选中结果

#### 2.1.2 筛选功能
- **菜系筛选**（多选）：
  - 川菜、渝菜、徽菜（皖）、西餐、新加坡餐、其他
- **用餐时段**（单选）：
  - 早餐、午餐、晚餐、夜宵
- **食材类型**（多选）：
  - 荤菜、素菜、汤、粥、煲、饭、米饭、面食、杂粮
- **组合筛选**：支持同时选中多个筛选项，筛选结果取并集

#### 2.1.3 食材库
- 支持成千上万个食材数据
- 每个食材可属于多个类别（无需去重）
- 多级分层结构：菜系 → 具体菜品

### 2.2 用户交互

#### 2.2.1 选择流程
1. 用户设置筛选条件
2. 点击"开始选择"按钮
3. 系统从符合条件的食材中随机抽取一款
4. 展示选中结果
5. 可选择"重新选择"或"修改条件"

#### 2.2.2 界面交互
- 筛选条件以标签/按钮组形式展示
- 选中状态视觉反馈
- 随机选择动画效果
- 结果展示卡片

## 3. 数据结构

### 3.1 食材数据格式
```json
{
  "id": "unique_id",
  "name": "食材名称",
  "categories": {
    "cuisine": ["川菜", "渝菜"],
    "mealType": ["午餐", "晚餐"],
    "foodType": ["荤菜", "米饭"]
  }
}
```

### 3.2 分类层级
```
├── 川菜
│   ├── 肉类
│   │   ├── 回锅肉
│   │   ├── 麻婆豆腐
│   │   └── 水煮鱼
│   └── 素菜
│       ├── 鱼香茄子
│       └── 干煸四季豆
├── 渝菜
│   └── ...
├── 徽菜(皖)
│   └── ...
├── 西餐
│   └── ...
├── 新加坡餐
│   └── ...
└── 其他
    └── ...
```

## 4. 多平台架构

### 4.1 网页版
- 单页应用（SPA）
- 响应式设计，适配桌面和移动端
- 本地食材库JSON文件

### 4.2 小程序版
- 独立小程序项目
- 原生或Taro/uni-app跨平台框架
- 共享同一食材库数据结构

## 5. 验收标准

### 5.1 功能验收
- [ ] 可从10000+食材中随机选择
- [ ] 菜系筛选正确（川渝皖西餐新加坡餐）
- [ ] 用餐时段筛选正确（早中晚餐夜宵）
- [ ] 食材类型筛选正确（荤素汤粥煲米饭面食杂粮）
- [ ] 多条件组合筛选正常
- [ ] 无需去重逻辑（同一食材可出现在多个分类）
- [ ] 随机选择结果不重复上次选择

### 5.2 性能验收
- [ ] 页面加载 < 2秒
- [ ] 筛选响应 < 500ms
- [ ] 选择操作响应 < 1秒

### 5.3 兼容性验收
- [ ] 网页版：Chrome、Safari、Firefox、Edge最新版本
- [ ] 小程序版：微信小程序
