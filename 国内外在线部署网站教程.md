## 国外免费部署平台

### 1. Netlify ⭐⭐⭐ 最推荐

**易用性：** ⭐⭐⭐⭐⭐  
**知名度：** ⭐⭐⭐⭐⭐  
**适合人群：** 所有用户，特别是新手

#### 优点
- ✅ 拖拽即可部署，无需任何配置
- ✅ 自动 HTTPS，全球 CDN
- ✅ 支持自定义域名
- ✅ 与 GitHub/GitLab 无缝集成，自动部署
- ✅ 免费额度充足（100GB 带宽/月）
- ✅ 表单处理、函数支持等高级功能

#### 缺点
- 国内访问速度一般（无国内节点）

#### 核心部署步骤

**方法A：拖拽部署（最简单）**
```bash
1. 访问 https://www.netlify.com 注册账号
2. 登录后进入 "Sites" 页面
3. 将整个项目文件夹拖拽到部署区域
4. 等待30秒，获得随机生成的网址
5. 完成！
```

**方法B：GitHub 连接（推荐长期使用）**
```bash
1. 将代码推送到 GitHub 仓库
2. 在 Netlify 点击 "Add new site" -> "Import an existing project"
3. 选择 GitHub，授权并选择仓库
4. 保持默认设置，点击 "Deploy site"
5. 每次推送代码自动更新
```

**自定义站点名称：**
```
Site settings -> Change site name -> 输入名称（如 eatwhat-lin）
访问地址变为：https://eatwhat-lin.netlify.app
```

### 3. GitHub Pages ⭐⭐ 经典选择

**易用性：** ⭐⭐⭐⭐  
**知名度：** ⭐⭐⭐⭐⭐  
**适合人群：** 已有 GitHub 账号的用户

#### 优点
- ✅ 完全免费，无带宽限制
- ✅ 与代码仓库深度集成
- ✅ 支持自定义域名
- ✅ 自动 HTTPS
- ✅ 稳定性极高

#### 缺点
- 需要手动配置（相比 Netlify/Vercel）
- 国内访问速度较慢
- 不支持服务端功能

#### 核心部署步骤

```bash
# 1. 创建 GitHub 仓库
# 访问 https://github.com/new 创建公开仓库

# 2. 推送代码
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/eatwhat.git
git push -u origin main

# 3. 启用 Pages
# 进入仓库 Settings -> Pages
# Source 选择 "main branch"
# 点击 Save

# 4. 等待1-2分钟，访问：
# https://YOUR_USERNAME.github.io/eatwhat/
```

### 4. Cloudflare Pages ⭐⭐ 速度快

**易用性：** ⭐⭐⭐  
**知名度：** ⭐⭐⭐⭐  
**适合人群：** 追求访问速度，有一定技术基础

#### 优点
- ✅ 基于 Cloudflare 全球 CDN，速度快
- ✅ 国内访问优于 GitHub Pages
- ✅ 免费额度大（无限带宽）
- ✅ 自动 HTTPS
- ✅ 支持自定义域名

#### 缺点
- 配置相对复杂
- 必须使用 Git 仓库
- 需要选择正确的构建配置

#### 核心部署步骤

```bash
# 1. 准备 GitHub 仓库（必须先推送代码）
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/eatwhat.git
git push -u origin main

# 2. 访问 https://pages.cloudflare.com 注册/登录

# 3. 创建 Pages 项目
# Workers & Pages -> Create application -> Pages
# Connect to Git -> 选择 GitHub 仓库

# 4. 关键配置（重要！）
# - Project name: eatwhat
# - Production branch: main
# - Framework preset: None（必须选 None！）
# - Build command: 留空（静态网站不需要）
# - Build output directory: 留空或填 "."

# 5. 点击 "Save and Deploy"
# 等待1-2分钟，获得 *.pages.dev 网址
```

**注意：** 项目中需包含 `_redirects` 文件以确保路由正确。

## 常见问题 FAQ

### Q1: 为什么我的网站显示空白？
**A:** 检查浏览器控制台（F12）：
- 如果是 404 错误，确认文件路径正确
- 检查文件名大小写（Linux 服务器区分大小写）
- 确认所有资源文件（CSS、JS、图片）都已上传

### Q2: 如何更新网站内容？
**A:** 
- **Netlify/Vercel/GitHub/Cloudflare**: 推送代码到 Git 仓库，自动更新
- **Gitee Pages**: 推送代码后，进入 Pages 设置点击"更新"
- **拖拽部署**: 重新拖拽整个文件夹

### Q3: 自定义域名如何配置？
**A:** 
1. 在部署平台添加自定义域名
2. 在域名服务商处添加 DNS 记录（CNAME 或 A 记录）
3. 等待 DNS 生效（通常几分钟到几小时）
4. 国内平台需要 ICP 备案

### Q4: 如何选择最适合的平台？
**A:** 
- **新手/快速测试**: Netlify
- **长期项目**: Cloudflare Pages 或 GitHub Pages
- **国内用户为主**: Gitee Pages
- **React/Next.js 项目**: Vercel
- **企业级应用**: 阿里云/腾讯云

### Q5: 免费额度够用吗？
**A:** 对于个人项目和小流量网站，所有平台的免费额度都足够。只有当月访问量超过数万时才需要考虑升级。
