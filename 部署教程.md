# "吃什么" 项目部署教程

本文档提供三个免费平台的详细部署步骤，选择任意一个即可完成部署。

---

## 方案一：Netlify（最简单 ⭐⭐⭐）

**适合人群：** 完全新手，想要最快部署
**预计时间：** 5-10分钟
**优点：** 拖拽即可部署，无需命令行

### 步骤1：注册账号

1. 访问 [https://www.netlify.com](https://www.netlify.com)
2. 点击 "Sign up"
3. 选择以下任一方式注册：
   - GitHub 账号（推荐）
   - GitLab 账号
   - Bitbucket 账号
   - 邮箱注册

### 步骤2：部署网站

#### 方法A：拖拽部署（最简单）

1. 登录后，点击 "Sites" 标签
2. 找到页面底部的拖拽区域
3. 将整个 `eatWhat` 文件夹拖拽到该区域
4. 等待上传完成（约30秒）
5. 部署成功！你会看到一个随机生成的网址

#### 方法B：GitHub 连接部署（推荐）

1. 登录后，点击 "Add new site" -> "Import an existing project"
2. 选择 "GitHub"
3. 授权 Netlify 访问你的 GitHub
4. 选择你的仓库（如果还没有，先创建 GitHub 仓库）
5. 保持默认设置，点击 "Deploy site"
6. 等待部署完成

### 步骤3：自定义域名（可选）

1. 进入 "Site settings"
2. 点击 "Domain management"
3. 点击 "Set up a custom domain"
4. 输入你想要的域名
5. 按照提示配置 DNS

### 步骤4：修改站点名称（可选）

1. 进入 "Site settings"
2. 点击 "Change site name"
3. 输入新名称，如 `eatwhat-lin`
4. 保存后访问地址变为：`https://eatwhat-lin.netlify.app`

### 常见问题

**Q: 如何更新网站内容？**
- 方法A：重新拖拽整个文件夹
- 方法B：如果使用 GitHub，推送代码后自动更新

**Q: 为什么图片不显示？**
- 确保图片路径是相对路径
- 检查文件名大小写是否正确

**Q: 访问速度慢怎么办？**
- Netlify 全球 CDN，通常很快
- 可以考虑使用自定义 CDN

## 方案二：Cloudflare Pages（速度快 ⭐⭐）

**适合人群：** 追求访问速度，有一定技术基础
**预计时间：** 15-20分钟
**优点：** 基于 Cloudflare CDN，国内访问较快

### 步骤1：注册 Cloudflare 账号

1. 访问 [https://pages.cloudflare.com](https://pages.cloudflare.com)
2. 点击 "Sign up"
3. 使用邮箱注册
4. 验证邮箱

### 步骤2：准备 GitHub 仓库

Cloudflare Pages 需要连接 Git 仓库，所以需要先上传代码到 GitHub：

```bash
# 1. 在 GitHub 创建新仓库
# 访问 https://github.com/new
# 仓库名：eatwhat
# 设为 Public

# 2. 本地初始化并推送
cd c:\Users\lin\Downloads\eatWhat

git init
git add .
git commit -m "Initial commit"

git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/eatwhat.git
git push -u origin main
```

### 步骤3：连接 Cloudflare Pages

1. 登录 Cloudflare Dashboard
2. 左侧菜单选择 "Workers & Pages"
3. 点击 "Create application" -> "Pages"
4. 点击 "Connect to Git"
5. 选择 GitHub，授权访问
6. 选择你刚才创建的 `eatwhat` 仓库
7. 配置构建设置：
   - **Production branch**: `main`
   - **Build command**: 留空（静态网站不需要）
   - **Build output directory**: 留空或填 `/`
8. 点击 "Save and Deploy"

### 步骤4：等待部署完成

1. 部署通常需要 1-2 分钟
2. 完成后会显示一个 `*.pages.dev` 的网址
3. 点击即可访问你的网站

### 步骤5：自定义域名（可选）

1. 进入 Pages 项目设置
2. 点击 "Custom domains"
3. 点击 "Set up a custom domain"
4. 输入你的域名
5. Cloudflare 会自动配置 DNS

### 高级配置

#### 添加 _redirects 文件（可选）

在项目根目录创建 `_redirects` 文件：

```
/*    /index.html   200
```

这样可以确保刷新页面时不会 404。

#### 配置环境变量（可选）

1. 进入项目设置
2. 点击 "Environment variables"
3. 添加你需要的变量

### 常见问题

**Q: 部署失败怎么办？**
- 检查构建命令是否正确
- 查看部署日志找出错误
- 确保所有文件都在仓库中

**Q: 如何回滚到之前的版本？**
- 进入 "Deployments" 标签
- 找到要回滚的版本
- 点击 "..." -> "Retry deployment"

**Q: 国内访问速度如何？**
- Cloudflare 有全球节点
- 国内访问比 GitHub Pages 快
- 但不如国内平台

## 方案三：Gitee Pages（国内最快 ⭐）

**适合人群：** 主要面向国内用户
**预计时间：** 20-30分钟（含认证时间）
**优点：** 国内访问极快，备案可选

### ⚠️ 重要提示

Gitee Pages 需要实名认证，且目前对新用户有限制。如果无法使用，建议选择前两个方案。

### 步骤1：注册 Gitee 账号

1. 访问 [https://gitee.com](https://gitee.com)
2. 点击 "注册"
3. 填写手机号、验证码等信息
4. 完成实名认证（必须）

### 步骤2：创建仓库

1. 登录后，点击右上角 "+" -> "新建仓库"
2. 填写信息：
   - **仓库名称**: `eatwhat`
   - **是否开源**: 公开
   - **初始化仓库**: 不勾选
3. 点击 "创建"

### 步骤3：上传代码

#### 方法A：使用 Git 命令行

```bash
cd c:\Users\lin\Downloads\eatWhat

git init
git add .
git commit -m "Initial commit"

git remote add origin https://gitee.com/YOUR_USERNAME/eatwhat.git
git push -u origin master
```

#### 方法B：网页上传

1. 进入仓库页面
2. 点击 "上传文件"
3. 拖拽所有文件到上传区域
4. 填写提交信息
5. 点击 "提交"

### 步骤4：启用 Gitee Pages

1. 进入仓库页面
2. 点击顶部 "服务" -> "Gitee Pages"
3. 首次使用需要等待审核（可能几小时到几天）
4. 审核通过后：
   - **分支**: 选择 `master` 或 `main`
   - **目录**: 留空
   - **强制 HTTPS**: 勾选
5. 点击 "启动"

### 步骤5：访问网站

部署完成后，访问地址格式：
```
https://YOUR_USERNAME.gitee.io/eatwhat/
```

### 常见问题

**Q: 为什么找不到 Gitee Pages 选项？**
- 确认已完成实名认证
- 新用户可能需要等待一段时间才能使用
- 某些时期 Gitee 会暂停新用户的 Pages 服务

**Q: 审核需要多久？**
- 通常 1-3 个工作日
- 有时更快，取决于审核队列

**Q: 如何更新网站？**
1. 推送新代码到仓库
2. 进入 Gitee Pages 设置
3. 点击 "更新" 按钮

**Q: 可以自定义域名吗？**
- 可以，但需要备案
- 进入 Pages 设置 -> 自定义域名

## 快速开始（推荐流程）

### 如果你想最快上线：

```
1. 注册 Netlify (2分钟)
2. 拖拽文件夹部署 (1分钟)
3. 获得网址，分享给朋友 (瞬间)
```

### 如果你想长期维护：

```
1. 创建 GitHub 仓库 (3分钟)
2. 推送代码 (2分钟)
3. 连接 Cloudflare Pages (5分钟)
4. 配置自定义域名（可选）(10分钟)
总耗时：约 20 分钟
```

### 如果主要面向国内用户：

```
1. 注册 Gitee 并认证 (10分钟)
2. 创建仓库并上传代码 (5分钟)
3. 申请 Pages 服务 (等待审核)
4. 审核通过后启动 (1分钟)
总耗时：30分钟 + 审核等待时间
```
