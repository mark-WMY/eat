# 继续向食材数据库添加条目

## 需求概述

根据 skill 文件规则，继续向 `js/ingredients_common.json` 和 `js/ingredients_full.json` 添加菜品条目。

## 规则

1. **同步添加**: common 中添加的条目必须同步添加到 full 中（common 是 full 的子集）
2. **真实存在**: 所有条目必须是真实存在的菜品，禁止虚构/AI 生成
3. **≥3省覆盖**: common 的条目必须是全国超过三个省份都能找到/可轻松购买的菜品
4. **full 可接收**: 地域性强的真实菜品仅加入 full

## 添加策略（三批次）

### 批次A：全国常见新条目 → common + full

从 `full` 中筛选出全国性常见菜品，**promote 到 common**。同时新增尚不存在的全国性菜品到两者。

**筛选标准**：
- 八大菜系（川鲁粤苏浙闽湘徽）中无城市前缀的经典菜品 → 全国知名
- 其他菜系中已广泛全国化的菜品（东北菜、西北菜、湖北菜等）
- 全国连锁品类（沙县小吃品类、兰州拉面品类）
- 全国常见外国菜（已在各大城市普及）
- 通用主食/饮品

### 批次B：地域性强的新条目 → 仅 full

从 `full` 中确认已存在但地域性强的条目，**不加到 common**。新增的地域性真实菜品仅加入 full。

**筛选标准**：
- 城市/县前缀的菜品
- 极小众地方特色
- 外国小众菜品

### 批次C：尚未存在的全国性菜品 → 新增到 common + full

发现数据库中尚未收录但确实在全国常见的菜品，新增到两个文件。

## 受影响文件

| 文件 | 操作 |
|------|------|
| `js/ingredients_common.json` | 添加 promote 的条目和新条目 |
| `js/ingredients_full.json` | 添加新条目（promote 的条目已在 full 中） |

## 实施细节

### 使用 Python 脚本操作

利用已有的 `scripts/ingredient_tools.py` 中的工具函数：
- `load_data()` / `save_data()` - 读写 JSON
- `get_existing_names()` - 获取已有名称集合
- `get_next_id()` - 生成新 ID
- `add_items_safe()` - 安全添加

**备注**：实际添加前需逐一比对确认是否已存在，避免重复。

## 数据流

1. 加载 common.json 和 full.json
2. 提取 common 中已有名称集合
3. 按上述策略分类处理条目
4. ID 从当前最大 ID 往后递增
5. 对于 promote 条目：将 full 中的条目对象复制到 common
6. 对于全新条目：创建新条目对象（包含 categories, attributes）
7. 验证 common 仍然是 full 的子集
8. 保存两个文件

## 边界条件和异常处理

- **重复检查**：添加前按 name 去重
- **ID 冲突**：使用 `get_next_id()` 确保 ID 唯一
- **类别验证**：确认 categories 值在预定义范围内
- **JSON 合法性**：保存后可用 `json.load()` 重新读取验证
- **子集验证**：验证 common 中所有 name 都在 full 中存在

## 预期结果

- common 条目数增加（promoted + 新增）
- full 条目数增加（新增）
- common 仍然是 full 的子集
- 所有新条目都是真实存在的
- common 中的条目均为全国≥3省常见的菜品
